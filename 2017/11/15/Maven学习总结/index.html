Maven学习总结

发表于 2017-11-15 | 分类于 Maven

很早之前为了学习框架，草草的就学习了Maven，但是只是会构建一个简单的web项目，并没有深入学习过Maven。这次学过去，有很多新的东西以前并没有接触过。
总结一下。

我们为什么使用Maven？

从一开始学习JavaWeb的时候，大家都推荐使用Maven来构建项目。那个时候我很疑惑，不就是管理jar包而已，用的着这么麻烦？而现在我才逐渐了解Maven的优势。

  • 管理jar包依赖，不会出现重复多余的jar包，和版本冲突的问题。
  • 完整的构建项目流程，从清理，编译，在到后期的打包，发布流程一体化。
  • 有很多的plugin可以扩展功能。

如何使用Maven？

安装Maven注意事项

关于配置环境变量：

  • 在配置环境变量的时，注意将Maven文件夹的路径,配置成MAVEN_HOME。
    再将MAVEN_HOME下的bin添加到path中，就可以在DOS下使用mvn的命令了。
  • 还有注意将Jdk中的JAVA_HOME配置成jdk文件夹路径。
    再将JAVA_HOME下的bin添加到path中，否则在使用Maven中会提示找不到jre的错误

关于Setting.xml:

Setting.xml是很重要的配置文件，在之前并没有注意。它位于conf文件夹中，注意修改配置文件Setting.xml中的参数。

1.修改镜像仓库，Maven默认镜像仓库的地址在国外，在国内访问的速度会很慢。

2.修改本地仓库默认路径，Maven仓库的默认路径是在来在C:\Users\用户\.m2。可以修改配置
更改为本地仓库。

3.修改Maven默认的JDK版本,Maven默认的JDK版本是1.5，每次修改太过麻烦，直接修改配置~

pom.xml文件详解

pom.xml文件(project object model)，项目对象模型。Maven项目中最重要的xml文件。

Maven生命周期

  • clean清理项目
    pre-clean：清理前，clean清理项目，post-clean清理后
  • default构建项目
    compile，test，package，install
  • site 生成项目站点
    pre-site：生成站点前，site：生成站点，post-site：生成站点后，site-deploy：发布生成站点到服务器上

消除依赖传递

依赖传递，顾名思义是与有传递的关系在。
A -> B -> C
A项目依赖B项目，而B项目又依赖于C项目
这个时候在项目A中其实存在着项目C的jar包，但是这些jar包有时候又不需要。
这个时候可以在A的pom文件中对B的依赖使用<exclusion>标签来消除传递依赖

之后A中就不会有C里的slf4j了~

依赖冲突

Maven中也会发生一定的依赖冲突问题，Maven有它自己的解决方案。

  1. 短路优先策略

    A -> B -> C -> x.jar
    A -> D -> x.jar
    同样是俩条依赖路径，Maven会默认从路径短的依赖来进行继承，会优先继承D下的x.jar。

  2. 先声明先优先策略
    A -> B -> x.jar
    A -> D -> x.jar
    对于长度相同的依赖路径，Maven会自动在A的pom文件中进行筛选，看B和D那个写在前面，如果是B写在前面会优先继承B下的x.jar，D同理。

聚合和继承

在Maven中还可以使用一个Maven项目对其中的所有子模块同时进行编译。
在项目的pom加入<modules>来引入A，B，C子模块。
