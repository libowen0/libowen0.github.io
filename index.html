Spring学习笔记（2）--IOC基础和基本原理

发表于 2017-10-24

1. Ioc

IOC (Inversion of control)是一种设计思想，在Java开发中，意味着将对象交给容器控制
并且对象之间的依赖关系也交给容器来管理。

优势：解耦，传统方式都是在类的内部创建对象，使得对象于对象间的耦合程度较高，不利于测试。而控制反转后的对象都是经过容器来创建，需要的时候注入的，而这一切都是在框架内部完成的。

2. Ioc和DI

DI 依赖注入，指得是容器动态的将依赖关系注入到组件中属于IOC容器的功能。

3. 详解Ioc容器

Spring Ioc容器的代表的是org.springframework.beans包中的BeanFactory接口，提供了最基本功能
而实际开发使用的是org.springframework.context包下的ApplicationContext接口，他在基础之上扩展了功能用于企业级应用。ApplicationContext完全继承BeanFactory，因而BeanFactory所具有的语义也适用于ApplicationContext。

Spring学习笔记(1)--Spring概述

发表于 2017-10-24

本系列是在学习Spring框架中的总结和记录
推荐开涛大神的博客系列跟我学spring3

1. Spring是什么？

Spring 是开源轻量级JavaEE框架，用于简化企业级应用程序开发。

2. Spring能帮我们做什么

  1. 传统开发，对象间的依赖关系由程序内部创建，加大了对象间的耦合程度，每次修改都需要重新编译，降低效率，而且不利于测试。
    Spring 内部的Ioc容器，可自动管理创建的Bean，依赖注入关系依照配置文件来组装。
  2. 传统开发在一些如，日志记录，权限控制大部分代码是重复的，每次添加功能都需要很多的重复代码 。
    Spring 面向切面编程能帮助我们无耦合的实现日志记录，性能统计，安全控制。
  3. 事务管理的简化，在传统数据库事务管理，每次都需要如“获取链接，执行SQL，提交或回滚事务，关闭链接”等一系列机械式的代码。
    Spring能非常简单的帮我们管理数据库事务。只需要获取链接执行SQL。
  4. Spring提供与第三方数据访问框架的无缝集成,而且自己也提供了一套JDBC访问模版，方便访问数据库。
  5. Spring还提供与第三方Web（如Struts、JSF）框架无缝集成，而且自己也提供了一套Spring MVC框架，来方便web层搭建。
  6. Spring能方便的与Java EE（如Java Mail、任务调度）整合，与更多技术整合（比如缓存框架）。

3. Spring 基础

3.1 Spring架构图

核心容器：Core、Beans、Context、EL模块

  • Core模块：封装了框架的最底层部分，包括资源访问、类型转换及常用工具类。
  • Beans模块：提供了框架的基础部分，包括控制反转和依赖注入。其中BeanFactory是容器核心，本质是“工厂设计模式”的实现，且无序编程实现“单例设计模式”，单例完全由容器控制，而且提倡面向接口编程，而非面向实现编程。所有依赖关系交给框架管理。
  • Context模块：以Core和Beans为基础，集成Beans模块功能并添加资源绑定、数据验证、国际化
    、JavaEE支持、容器生命周期、事件传播等；核心接口是ApplicationContext。
  • EL模块:提供强大的表达式语言支持，支持访问和修改属性值，方法调用，支持访问及修改数组、容器和索引器，命名变量，支持算数和逻辑运算，支持从Spring 容器获取Bean，它也支持列表投影、选择和一般的列表聚合等。

3.2 典型应用场景

Spring可以应用到许多场景，从最简单的标准Java SE程序到企业级应用程序都能使用Spring来构建。

典型Web应用程序应用场景：

在Web应用程序应用场景中，典型的三层架构，数据模型层实现域对象，数据访问层实现数据访问，
逻辑层实现业务逻辑，web层提供页面展示。所有组件都是由Spring进行管理，享受到Spring事务和AOP的好处，而且请求的唯一入口就是DispatcherServlet,它通过把请求映射到相应的web层组件来实现相应的请求功能。
