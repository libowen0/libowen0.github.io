什么是ajax

Ajax 即“Asynchronous Javascript And XML”（异步 JavaScript 和 XML），是指一种创建交互式网页应用的网页开发技术。
Ajax = 异步 JavaScript 和 XML（标准通用标记语言的子集）。
Ajax 是一种用于创建快速动态网页的技术。
Ajax 是一种在无需重新加载整个网页的情况下，能够更新部分网页的技术。
通过在后台与服务器进行少量数据交换，Ajax 可以使网页实现异步更新。这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新。
传统的网页（不使用 Ajax）如果需要更新内容，必须重载整个网页页面。

这是百度对它的定义，足够详细。
值得补充的一点是对异步的理解，异步是相对于同步来说的，在这里他们指的是服务器和浏览器的交互模式。
同步，每次请求发出之后，用户操作即被阻塞，必须要求返回响应后继续操作。而异步指的是发出请求后，用户无需等待响应，一切由ajax来实现，无需进行刷新网页就可以局部更新数据。提高了俩端的沟通效率。

整合方式

俩种不同的整合方式 ：原始dao方式，和官方推荐的mapper接口方式。
相比原始dao的方式，我更偏向于使用mapper接口。

有俩个理由：

  • 约定大于配置的方式，通过对框架使用的一定限制来提高程序的健壮性，减少大量代码。
  • 面向接口式编程，dao层就是接口层，而dao层的实现由框架通过代理来创建。

整合思路

完全去掉SqlMapConfig.xml文件，把所有配置放在Spring中。

  1. 配置数据源（JDBC，c3p0等）
  2. 配置SqlSessionFactory
    • 引入数据源
    • 配置类的别名为批量别名定义
    • 配置映射类型
  3. 配置批量加载mapper

Spring学习笔记（2）--IOC基础和基本原理

发表于 2017-10-24

1. Ioc

IOC (Inversion of control)是一种设计思想，在Java开发中，意味着将对象交给容器控制
并且对象之间的依赖关系也交给容器来管理。

优势：解耦，传统方式都是在类的内部创建对象，使得对象于对象间的耦合程度较高，不利于测试。而控制反转后的对象都是经过容器来创建，需要的时候注入的，而这一切都是在框架内部完成的。

2. Ioc和DI

DI 依赖注入，指得是容器动态的将依赖关系注入到组件中属于IOC容器的功能。

Spring学习笔记（1）--Spring概述

发表于 2017-10-24

本系列是在学习Spring框架中的总结和记录
推荐开涛大神的博客系列跟我学spring3

1. Spring是什么？

Spring 是开源轻量级JavaEE框架，用于简化企业级应用程序开发。

2. Spring能帮我们做什么

  1. 传统开发，对象间的依赖关系由程序内部创建，加大了对象间的耦合程度，每次修改都需要重新编译，降低效率，而且不利于测试。

Spring 内部的Ioc容器，可自动管理创建的Bean，依赖注入关系依照配置文件来组装。

  1. 传统开发在一些如，日志记录，权限控制大部分代码是重复的，每次添加功能都需要很多的重复代码 。
    Spring 面向切面编程能帮助我们无耦合的实现日志记录，性能统计，安全控制。
  2. 事务管理的简化，在传统数据库事务管理，每次都需要如“获取链接，执行SQL，提交或回滚事务，关闭链接”等一系列机械式的代码。
    Spring能非常简单的帮我们管理数据库事务。只需要获取链接执行SQL。
  3. Spring提供与第三方数据访问框架的无缝集成,而且自己也提供了一套JDBC访问模版，方便访问数据库。
  4. Spring还提供与第三方Web（如Struts、JSF）框架无缝集成，而且自己也提供了一套Spring MVC框架，来方便web层搭建。
  5. Spring能方便的与Java EE（如Java Mail、任务调度）整合，与更多技术整合（比如缓存框架）。
